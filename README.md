# 🚀 개발자 번아웃 조기 경보 시스템 (Burnout Early Warning System)
> **SWCON372 응용데이터분석 - 데이터 기반 자기 객관화 및 생산성 관리 프로젝트**

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 현대 개발 환경에서 심화되는 **'인지적 과부하'**를 행동 로그 분석을 통해 사전에 탐지하고, 개발자에게 **'자기 객관화'** 도구를 제공하여 번아웃을 예방하는 시스템을 구축합니다.

---

## 2. 프로젝트 배경 및 필요성 (Necessity)

### 📊 개발자 생산성의 역설
2026년 현재, AI 코딩 어시스턴트의 발전으로 단순 구현 속도는 비약적으로 향상되었습니다. 그러나 개발자는 더 짧은 시간 안에 더 많은 아키텍처적 의사결정을 내려야 하는 상황에 직면했으며, 이는 심각한 **인지적 과부하**로 이어지고 있습니다.

---

## 3. 핵심 구현 성과 (Implementation Milestones - 2026.03.25)

오늘 프로젝트의 핵심 데이터 수집 도구인 **VS Code Extension (Burnout Detector)**의 주요 기능 구현 및 배포 준비를 마쳤습니다.

### ✅ 구현된 4대 핵심 지표 (Core Metrics)
1. **타이핑 속도 변동성 분석**: 초 단위로 정규화된 타이핑 빈도(`count`)를 기록하여 데이터 효율성을 높이고 속도 저하를 감지합니다.
2. **에러 해결 리드타임 측정**: `onDidChangeDiagnostics`를 통해 에러 발생부터 해결까지의 소요 시간을 ms 단위로 정밀 측정합니다.
3. **에러 유형 자동 분류**: 에러 메시지를 분석하여 단순 오타/문법 실수(`Basic`)와 복잡한 논리 오류(`Complex`)를 구분하여 인지적 상태를 추론합니다.
4. **수정 반복 패턴(Undo/Redo) 추적**: `TextDocumentChangeReason`을 활용하여 특정 구간에서의 시행착오 빈도를 수집합니다.

### 📦 배포 및 수집 준비 상태 (Deployment Readiness)
- **VSIX 패키징 완료**: `vsce`를 활용한 오프라인 설치 파일(`.vsix`) 생성을 성공하여 지인 및 소규모 그룹 배포가 가능해졌습니다.
- **데이터 익명화 및 동의 로직**: 모든 데이터는 랜덤 생성된 `userId`와 함께 수집되며, 최초 실행 시 사용자의 명확한 **수집 동의**를 받는 팝업 로직을 포함합니다.
- **중앙 서버 전송 구조**: 로컬 로그(`burnout_log.json`)를 서버로 전송하기 위한 업로드 아키텍처를 설계했습니다.

---

## 4. 수집 행동 데이터 상세 명세 (Data Specification)

| 수집 범주 | 구체적 데이터 항목 (Fields) | 분석 목적 |
| :--- | :--- | :--- |
| **타이핑 (Typing)** | `timestamp`, `count` (1초 단위 묶음) | 전반적인 작업 속도 및 몰입도 저하 감지 |
| **디버깅 (Diagnostic)** | `duration`, `type` (Basic/Complex), `message` | 인지적 피로도에 따른 해결 능력 변화 측정 |
| **작업 흐름 (Workflow)** | `undo`, `redo` 발생 시각 및 빈도 | 판단력 저하로 인한 수정 반복 행위 탐지 |
| **세션 (Session)** | `userId` (익명화), `startTime`, `endTime` | 장기적인 행동 패턴 변화 및 개인별 기준점 설정 |

---

## 5. 향후 계획 (Future Works)
- **Marketplace 정식 등록**: 전 세계 개발자를 대상으로 한 대규모 데이터 크라우드 소싱 진행.
- **데이터 분석 모델 구축**: 수집된 행동 로그와 실제 번아웃 지수 간의 상관관계를 ML 모델로 학습.
- **실시간 피드백 대시보드**: 분석 결과를 시각화하여 개발자에게 휴식 권고 알림 제공.

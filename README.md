# 🚀 개발자 번아웃 조기 경보 시스템 (Burnout Early Warning System)
> **SWCON372 응용데이터분석 - 데이터 기반 자기 객관화 및 생산성 관리 프로젝트**

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 현대 개발 환경에서 심화되는 **'인지적 과부하'**를 행동 로그 분석을 통해 사전에 탐지하고, 개발자에게 **'자기 객관화'** 도구를 제공하여 번아웃을 예방하는 시스템을 구축합니다.

---

## 2. 프로젝트 배경 및 필요성 (Necessity)

### 📊 개발자 생산성의 역설
2026년 현재, AI 코딩 어시스턴트의 발전으로 단순 구현 속도는 비약적으로 향상되었습니다. 그러나 개발자는 더 짧은 시간 안에 더 많은 아키텍처적 의사결정을 내려야 하는 상황에 직면했으며, 이는 심각한 **인지적 과부하**로 이어지고 있습니다.

### 🧘 개인적 동기 및 목적
- **자기 객관화**: 주관적인 감정이 아닌, 객관적 데이터를 통해 자신의 컨디션을 파악합니다.
- **역량 유지**: 번아웃에 빠지기 전 스스로 상태를 점검하여 지속 가능한 개발 역량을 확보합니다.

---

## 3. 핵심 해결 과제 (Core Problem)

### 🖱️ 행동 로그 기반의 객관적 모니터링
단순 설문이 아닌 IDE(Integrated Development Environment) 내에서의 실질적인 행동 패턴을 로그화합니다.
- 에러 해결 시간 추이 분석
- 타이핑 리듬 및 코드 수정 패턴
- 반복적인 검색 키워드 및 빈도 로그화

### 🛡️ 선제적 예방을 통한 인재 이탈 방지
번아웃은 사후 약방문보다 **사전 감지**가 중요합니다. 위험 신호 발생 시 적절한 휴식 권고나 작업 난이도 조절 가이드를 제공하여 숙련된 인력의 이탈을 방지하는 인프라를 지향합니다.

### 📈 데이터 기반 의사결정
개발 효율의 수치적 변화를 확인하게 함으로써, 작업 우선순위를 재조정하고 최종적으로 고품질의 코드를 생산할 수 있는 환경을 조성합니다.

---

## 4. 분석 대상: 번아웃 전조 증상 (Early Symptoms)
본 프로젝트는 아래 3가지 핵심 지표를 데이터화하여 상관관계를 분석합니다.

1.  **타이핑 속도 변동성**: 평소 대비 타이핑 속도의 유의미한 저하 발생 여부.
2.  **디버깅 리드타임 증가**: 유사한 난이도의 에러를 해결하는 데 소요되는 시간의 점진적 증가.
3.  **반복적 검색 행위**: 동일하거나 유사한 기술적 문제를 해결하기 위해 검색창에 머무는 시간 및 반복 횟수.



---

## 5. 데이터 수집 및 분석 프로세스 (Data Collection & Analysis)

### 🛠️ Visual Studio Code Extension 활용
- **행동 로그 수집**: VS Code Extension을 개발하여 실제 개발 과정에서 발생하는 타이핑 리듬, 디버깅 시간, 검색 패턴 등의 행동 데이터를 백그라운드에서 수집합니다.
- **크라우드 소싱 (Crowdsourcing)**: VS Code Marketplace에 확장 프로그램을 등록하여, 다양한 환경의 개발자들로부터 익명화된 데이터를 대규모로 수집하는 연구 모델을 계획 중입니다.
- **실시간 분석 및 피드백**: 수집된 데이터를 기반으로 번아웃 위험도를 산출하여 개발자에게 시각화된 리포트와 예방 가이드를 실시간으로 제공합니다.
